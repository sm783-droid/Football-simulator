## Issue description

Manager stats (wins, draws, losses, points) are never updated after a match
result is recorded. While team stats update correctly via db_games._delta(),
the equivalent write to the managers table is missing. Manager records stay
at 0 permanently, regardless of match outcomes or score overrides.

Expected: manager stats mirror team stats at all times.
Actual: manager stats remain 0 after every result.

## Steps to reproduce

1. Register two teams (db.add_team)
2. Assign a manager to each team (db_managers.add_manager)
3. Add a fixture (db_games.add_fixture)
4. Save a score — e.g. 2-0 (db_games.save_score)
5. Read back the manager record (db_managers.get_manager)
6. Observe: mgr["points"] == 0 while team["points"] == 3

## Expected behavior

- After a win:  manager points == 3, wins == 1
- After a draw: manager points == 1, draws == 1
- After a loss: manager points == 0, losses == 1
- After a score override: manager stats revert the old result and apply
  the new one, staying in sync with the team (same as team stat revert logic)
