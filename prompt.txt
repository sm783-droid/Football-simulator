## Issue description

Manager stats (wins, draws, losses, points) are never updated when a match
result is saved. Team stats update correctly, but the matching write to the
manager record is absent. Manager stats stay at zero for every manager,
regardless of how many matches are played or what the scores are.

A second failure mode exists when a score is overridden: even if manager stats
were somehow updated on the first save, saving a new score over an existing
result must revert the old result and apply the new one. Stacking stats instead
of replacing them is also a bug.

## Steps to reproduce

1. Call `store.add_team("Home FC")` and `store.add_team("Away FC")`
2. Call `store.add_manager(home_id, "Alice")` and `store.add_manager(away_id, "Bob")`
3. Call `store.add_fixture(1, home_id, away_id)`
4. Call `store.save_score(game_id, 2, 0)` — a home win
5. Call `store.get_manager(home_id)` and inspect `mgr["points"]`
6. Observe: `mgr["points"] == 0` while `store.get_teams()[0]["points"] == 3`

## Expected behavior

Manager stats must mirror their team's stats at all times.

After a home win (e.g. 2–0):
- `home_mgr["wins"]   == 1`
- `home_mgr["points"] == 3`
- `away_mgr["losses"] == 1`
- `away_mgr["points"] == 0`

After a draw (e.g. 1–1):
- `home_mgr["draws"]  == 1`
- `home_mgr["points"] == 1`
- `away_mgr["draws"]  == 1`
- `away_mgr["points"] == 1`

After saving a second score that overrides the first (e.g. 2–0 then 0–2):
- Manager stats must reflect only the latest result — not the sum of both
- `home_mgr["wins"]   == 0`, `home_mgr["losses"] == 1`, `home_mgr["points"] == 0`
- `away_mgr["wins"]   == 1`, `away_mgr["losses"] == 0`, `away_mgr["points"] == 3`
